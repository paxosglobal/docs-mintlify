---
title: "identity_kyc_refresh"
description: "Event object returned for KYC refresh events"
---

This object is returned by the [Get Event API](/api-reference/events/get-event) when the event type is `identity.kyc_refresh.started`, `identity.kyc_refresh.completed`, or `identity.kyc_refresh.expired`.

## Object Fields

<ResponseField name="id" type="string" required>
Unique identifier for this object
</ResponseField>

<ResponseField name="identity_id" type="string" required>
The ID of the identity undergoing KYC refresh
</ResponseField>

<ResponseField name="status" type="string" required>
The current status of the KYC refresh process

**Available options:** `STARTED`, `COMPLETED`, `EXPIRED`
</ResponseField>

<ResponseField name="due_date" type="string">
When the KYC refresh must be completed (for started events only)
</ResponseField>

<ResponseField name="completed_date" type="string">
When the KYC refresh was completed (for completed events only)
</ResponseField>

<ResponseField name="expired_date" type="string">
When the KYC refresh expired (for expired events only)
</ResponseField>
