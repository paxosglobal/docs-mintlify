---
title: 'Make First API Request'
description: Use the access token to retrieve a list of Profiles.
---

Since every account has a `DEFAULT` [Profile](/api/v2#tag/Profiles), sending a request using [List Profiles](/api/v2#tag/Profiles/operation/ListProfiles) will always return a result, assuming the `funding:read_profile` scope was included when [adding scopes](/getting-started/authenticate#scopes) to the client.
Include the [access token](/getting-started/authenticate#token) from the authentication request in the authorization header.

### Sandbox

```shell
curl --location 'https://api.sandbox.paxos.com/v2/profiles?order=ASC&order_by=CREATED_AT' \
--header 'Accept: application/json' \
// highlight-next-line
--header 'Authorization: Bearer {access_token}'
```

### Production

```shell
curl --location 'https://api.paxos.com/v2/profiles?order=ASC&order_by=CREATED_AT' \
--header 'Accept: application/json' \
// highlight-next-line
--header 'Authorization: Bearer {access_token}'
```

A successful response includes all Profiles the client has access to.
Using the `?order=ASC&order_by=CREATED_AT` parameters ensures the `DEFAULT` Profile is first in the list.

```json
            "id": "7437da4d-c114-4a1f-8659-ceeaf868a1eb",

```

If you get an error, run through the [credentials](/getting-started/credentials) and [authentication](/getting-started/authenticate) guides and ensure your setup is correct.
If things are still not working, contact [Support](https://support.paxos.com).
